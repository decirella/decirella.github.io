Diagrams

February 3, 2022

Many times throughout the life of a project I find the need to create some visual component to go along with planning or communicating about some component of a system or technology. After coming to this conclusion, the next step is so often a question of what tool to use for the task! I have used several solutions that broadly fall into two categories: text-based and graphics-based.

On the text-based side I’ve been really enjoying using plantuml for creating various types of diagrams specified in text converted to an image. The full range of UML diagrams are supported along with an assortment of other types. The text specification also makes it possible to generate the diagram specification from another source (something I’m thinking about for a current project). The text-based nature also nice for version control.

The diagram above was generated with the text specification below:

/'
Title: Preservica Goobi Package/Ingest Scripts
Date: 20210420
'/

@startuml
title Preservica Goobi Package/Ingest Scripts
interface "Path to Goobi data" as IN
package "Preservica Goobi Packager" {
  [goobi_check] --> [goobi_pack] : goobi_mets path
  [goobi_pack]  --> [goobi_ingest] : (sip, workflowID, goobi)
  [goobi_ingest] 
}
node "Goobi" {
frame [Collection] {
  [Goobi Completed Object]
  }
}

cloud {
  [Preservica]
}

cloud {
  [Voyager]
}

cloud {
  [ArchivesSpace]
}

note right of [goobi_check] 
Monitors specificed directory 
for completed object
(completed can be presence of 
dcs.done file or other)
end note

note right of [goobi_pack] 
Takes input of object's goobi_mets.xml,
parses for descriptive record ID,
UUID values for file groups and files,
checksum values.

Arranges files for packaging
Calls SIPCreator with arguments for pacakge
Rewrites SIP XIP with Goobi UUIDs for
deliverable unit and files, verifies checksum

Output: SIP path, workflow context, Goobi path

end note

note right of [goobi_ingest] 
Takes input of SIP path, workflow context, Goobi path
Call Preservica API to start ingest workflow
Monitors ingest status
Writes preserica.done file to Goobi Object folder

end note

IN --> [goobi_check]
[goobi_pack] <-- [Voyager] : (MARCXML)
[goobi_pack] <-- [Goobi Completed Object]
[goobi_ingest] <--> [Preservica]
[goobi_ingest] --> [Goobi Completed Object] : preservica.done
[Preservica] <--> [ArchivesSpace] : Sync/Link
[goobi_check] --> Collection
@enduml
